The `order by` command is used to alphabetically order records. `desc` helps in descending order and `asc` helps in ascending order. The default ordering is `asc`.

### `asc`
**Example:**

``` SQL
 select * from employees order by last_name asc;
```

Here, we are ordering <a href="/blog/Introduction">`employees`</a> by `last_name` in ascending order.

**Result:**

<img src="/static/images/Screenshotfrom2023-11-0411-53-29.png" width="1500" />

In the above example, you can see that `last_name` is ordered in ascending order, starting from A.

### `desc`
**Example:**

``` SQL
select * from employees order by last_name desc;
```

`desc` will be in descending order.

<img src="/static/images/Screenshotfrom2023-11-0411-58-04.png" width="1500" />

`last_name` is in descending order, starting from W.

#### Keywords

### `asc`

```Elixir
HR.Repo.all(from c in HR.Employee, order_by: [asc: :last_name])
```

### `desc`

```Elixir
HR.Repo.all(from c in HR.Employee, order_by: [desc: :last_name])
```

In both queries, `c` is the reference variable referring to `HR.Employee`. To know about reference variable refer <a href="/blog/Aliases in Ecto">Aliases in Ecto</a>.
